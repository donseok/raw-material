# 철스크랩 수급 계획 및 구매 관리 시스템

## 개요

이 프로젝트는 철스크랩 구매 업무를 브라우저에서 시연할 수 있도록 만든 정적 웹 애플리케이션입니다.  
수급계획, 공장배분, 발주관리, 거래처관리, 수입계약, 재고현황, 구매실적을 탭 기반 대시보드로 제공합니다.

현재 기준으로는 "운영 제품"보다는 "업무 시나리오 검토용 프로토타입 + 일부 로컬 저장 기능"에 가깝습니다.

## 실행 방법

### 1. 로그인 페이지 열기

- 진입 파일: `login.html`
- 메인 화면인 `index.html`은 로그인 세션이 없으면 자동으로 `login.html`로 이동합니다.

### 2. 데모 계정

| 아이디 | 비밀번호 | 사용자명 |
| --- | --- | --- |
| `dongkuk1` | `1234` | 김철호 과장 |
| `dongkuk2` | `1234` | 박영수 대리 |

### 3. 주의사항

- 차트는 `Chart.js` CDN을 사용하므로 인터넷 연결이 필요합니다.
- 저장은 브라우저 `localStorage`, 로그인 상태는 `sessionStorage`를 사용합니다.

## 현재 구현 기능

### 1. 로그인

- 로그인 성공 시 `sessionStorage.loggedInUser`에 사용자 정보를 저장합니다.
- 메인 화면 상단에 로그인 사용자를 표시합니다.
- 로그아웃 시 세션을 삭제하고 로그인 화면으로 이동합니다.

### 2. 수급계획

- 월별 계획/국산/수입/실적 데이터를 표로 관리합니다.
- KPI 카드와 막대 차트를 자동 재계산합니다.
- 지원 기능:
  - 셀 더블클릭 편집
  - 엑셀 `Ctrl + V` 붙여넣기
  - 초기화
  - 브라우저 저장 복원
- 검증 규칙:
  - `plan`, `dom`, `imp`는 비울 수 없음
  - 음수 입력 불가
  - 붙여넣기 최대 12개월
  - 저장 데이터 구조 검증 후 복원

### 3. 공장배분

- 인천/포항 계획과 실적을 표로 관리합니다.
- KPI, 달성률, 진행률 바, 도넛 차트를 자동 갱신합니다.
- 지원 기능:
  - 셀 더블클릭 편집
  - 엑셀 `Ctrl + V` 붙여넣기
  - 초기화
  - 브라우저 저장 복원
- 검증 규칙:
  - 모든 값은 0 이상 숫자
  - 빈 값 저장 불가
  - 붙여넣기 최대 12개월

### 4. 발주관리

- 발주 목록 조회
- 신규 발주 등록
- 상단 KPI 자동 갱신
- PDF 발주서 미리보기 갱신
- 브라우저 저장 복원

현재는 입력된 발주를 `ordersData`에 저장하고, 가장 최근 발주를 PDF 미리보기 기준으로 사용합니다.

### 5. 거래처관리

- 거래처 목록 조회
- 신규 거래처 등록
- 거래처 비중 차트 즉시 반영
- 브라우저 저장 복원

등록된 거래처는 실시간으로 표에 반영되며, 거래처 비중 파이차트에도 바로 포함됩니다.

### 6. 수입계약

- 수입계약 목록 조회
- 신규 계약 등록
- KPI 재계산
- 상태별 차트 재계산
- 브라우저 저장 복원

### 7. 재고현황

- 인천/포항 공장 재고 조회
- 등급별 재고 차트 조회
- 상세 표 조회

이 탭은 현재 조회 전용입니다.

### 8. 구매실적

- 월별 구매 실적 조회
- 월별 추이 차트
- 등급별 비중 차트
- CSV 다운로드
- 인쇄

이 탭도 현재 조회 전용입니다.

| 키 | 설명 |
| --- | --- |

| `allocData` | 공장배분 데이터 |

## 화면 동작 흐름

### 초기 진입

1. `login.html` 또는 로그인 세션이 있으면 `index.html` 진입
2. `index.html`에서 CSS와 JS 파일 로드
3. `loadPlanFromStorage()`, `loadAllocFromStorage()`, `loadOrderDataFromStorage()`, `loadSupplierDataFromStorage()`, `loadImportDataFromStorage()` 실행
4. 조회형 탭 초기화
5. 수급계획/공장배분 초기 렌더링
6. 기본 차트 렌더링

### 탭 이동

1. 탭 버튼 클릭
2. 활성 탭 CSS 변경
3. 해당 탭 차트 재렌더링

### 데이터 입력

1. 모달 입력 또는 표 직접 편집
2. 숫자/구조 검증
3. 메모리 데이터 반영
4. `localStorage` 저장
5. 표/KPI/차트 재렌더링

## 현재 한계

- 서버 저장이 없습니다.
- 사용자 간 데이터 공유가 불가능합니다.
- 승인, 결재, 검색, 필터, 정렬 기능이 없습니다.
- 거래처/수입계약/발주 데이터는 로컬 브라우저 단위로만 유지됩니다.
- 로그인은 데모용 정적 계정 방식이며 실제 인증 시스템이 아닙니다.
- `inventory`, `stats`는 아직 조회 전용입니다.
- 화면과 로직이 분리돼 있지만, ES Module이 아니라 전역 함수 기반이라 스크립트 로딩 순서에 민감합니다.

## 개발 관점 메모

- 프레임워크 없이 순수 HTML/CSS/JavaScript로 작성되어 있습니다.
- 차트는 `Chart.js`로 탭 전환 시 재생성합니다.
- 데이터 정규화와 저장 검증은 각 탭별 JS 파일에서 처리합니다.
- 공통 저장/포맷/편집 로직은 `js/utils.js`에 있습니다.

## 요약

수급계획/공장배분/발주/거래처/수입계약은 로컬 저장 기반의 기본 입력 기능까지 포함하고 있습니다.  
재고현황과 구매실적은 조회 중심이고, 운영 시스템으로 발전시키려면 서버 저장과 인증 체계가 다음 핵심 단계입니다.
