# 바이브코딩 기초과정 강의 계획서

## 철스크랩 수급 계획 및 구매 관리 시스템 만들기

**대상**: 원료기획팀 50명 (코딩 경험 없음)
**시간**: 3시간 (이론 1시간 + 실습 2시간)
**강사**: 프로그래머 3명 (메인 강사 1명 + 보조 강사 2명)
**목표**: AI 트렌드를 이해하고, 바이브코딩으로 대시보드 웹 사이트가 만들어지는 과정을 체험

---

## 강의 형태

| 구분 | 방식 | 설명 |
|------|------|------|
| 1부 (1시간) | **강의** | AI 트렌드 소개 — 수강생은 듣고 이해 |
| 2부 (2시간) | **시연 + 따라하기** | 강사가 화면 공유하며 라이브 코딩, 수강생은 눈으로 확인하거나 직접 따라하기 선택 |

> 50명 전원이 직접 실습하면 환경 문제로 진행이 지연됩니다. **메인 강사가 빔프로젝터로 라이브 시연**하고, 수강생은 눈으로 따라가는 것을 기본으로 합니다. 직접 해보고 싶은 수강생은 서브 강사의 도움을 받아 따라합니다.

---

## 핵심 원칙

- **직접 타이핑 없음** — AI에게 한국어로 "이렇게 만들어줘"라고 설명하는 것이 전부
- **완성 경험 우선** — 2시간 시연으로 배포까지 완료된 결과물을 함께 확인
- **이 프로젝트를 축소 재현** — 전체가 아닌 핵심 3개 탭(수급계획, 발주관리, 재고현황)
- **우리 업무와 연결** — 원료기획팀이 실제 사용하는 철스크랩 데이터로 진행

## 1부: AI 트렌드 소개 (1시간)

### 오프닝 (5분)

- 강사 3명 소개
- 강의 구성 안내: "1부에서 AI 흐름을 이해하고, 2부에서 실제로 AI가 코드를 만드는 것을 함께 보겠습니다."

### AI 트렌드 개요 (20분)

#### 추천 구성

| 순서 | 주제 | 핵심 메시지 |
|------|------|------------|
| 1 | 생성형 AI란? | ChatGPT, Claude, Gemini — 텍스트/이미지/코드를 생성하는 AI |
| 2 | AI가 바꾸는 업무 방식 | 문서 작성, 데이터 분석, 보고서 생성 등 사무 업무 변화 |
| 3 | 제조업과 AI | 수요 예측, 품질 관리, 공급망 최적화 사례 |
| 4 | 코딩의 민주화 | 프로그래머가 아니어도 AI로 프로그램을 만들 수 있는 시대 |

> **포인트**: 원료기획팀 업무와 연결 — "우리가 매일 하는 수급계획, 발주, 재고 관리를 AI가 어떻게 도울 수 있는지"

### 바이브코딩이란? (20분)

#### 개념 설명

> "바이브코딩(Vibe Coding)은 Andrej Karpathy(테슬라 AI 책임자 출신)가 제안한 개념입니다. 프로그래밍 언어를 몰라도 AI에게 한국어로 '이렇게 만들어줘'라고 말하면 AI가 코드를 만들어줍니다."

#### 핵심 사이클

```
① 한국어로 지시 → ② AI가 코드 생성 → ③ 브라우저에서 확인 → ④ 수정 요청
```

#### 도구 소개

| 도구 | 설명 |
|------|------|
| **Cursor** | AI가 내장된 코드 편집기 (오늘 사용) |
| **Claude** | Anthropic의 AI 모델 (코드 생성 능력 우수) |
| **GitHub** | 코드 저장소 (구글 드라이브의 코드 버전) |
| **Vercel** | 웹 사이트 배포 플랫폼 (버튼 하나로 전 세계 공개) |

#### 사례 영상/스크린샷

- 간단한 프롬프트로 웹 페이지가 만들어지는 30초 시연
- "이런 걸 2부에서 처음부터 끝까지 해봅니다"

### 완성작 라이브 데모 (10분)

## 2부: 바이브코딩 라이브 시연 (2시간)

### 진행 방식

| 역할 | 담당 | 하는 일 |
|------|------|--------|
| **메인 강사** | 1명 | 빔프로젝터로 화면 공유, Cursor에서 라이브 시연, 설명 |
| **보조 강사** | 2명 | 강의실 좌/우 구역 나눠 배치, 따라하기 수강생 지원, 막힌 부분 해결 |

> 각 Step 시작 전에 "지금부터 ___를 만듭니다. 완성되면 이런 모습입니다"라고 **완성 스크린샷을 먼저** 보여줍니다.

### Step 1: 프로젝트 생성 + 로그인 페이지 (20분)

#### 1-1. Cursor 소개 및 프로젝트 폴더 생성 (5분)

메인 강사가 화면 공유하며:

1. Cursor 실행 — "이것이 AI가 내장된 코드 편집기입니다"
2. File → Open Folder → 바탕화면에 `my-dashboard` 폴더 생성 후 열기
3. 채팅창 열기 (Cmd+L / Ctrl+L) — "여기에 한국어로 지시합니다"

#### 1-2. 첫 번째 프롬프트 — 로그인 페이지 (15분)

Cursor 채팅창에 입력하는 모습을 보여주며:

```
동국제강 철스크랩 관리 시스템의 로그인 페이지를 만들어줘.

파일: login.html (단일 HTML 파일, CSS/JS 인라인)

요구사항:
- 남색(#1a237e) + 주황(#ff8f00) 컬러 테마
- 중앙 정렬 로그인 카드
- 아이디/비밀번호 입력 필드
- 로그인 버튼
- 계정 2개 하드코딩: dongkuk1/1234 (김철호 과장), dongkuk2/1234 (박영수 대리)
- 로그인 성공 시 sessionStorage에 사용자 정보 저장 후 index.html로 이동
- 로그인 실패 시 빨간색 오류 메시지 + 흔들기 애니메이션
- 이미 로그인된 상태면 index.html로 자동 이동
- 한국어 UI
```

**시연 포인트**:
- AI가 코드를 생성하는 과정을 실시간으로 보여줌
- Accept 클릭 → login.html 파일이 자동 생성되는 순간 강조
- 브라우저에서 열어 로그인 동작 확인

> **강사 멘트**: "제가 한 일은 '이렇게 만들어줘'라고 한국어로 쓴 것뿐입니다. 프로그래밍 언어를 하나도 몰라도 됩니다. 프롬프트가 곧 설계서입니다. 구체적으로 쓸수록 원하는 결과가 나옵니다."

---

### Step 2: 대시보드 뼈대 (30분)

#### 2-1. 기본 구조 프롬프트 (20분)

```
이제 메인 대시보드 index.html을 만들어줘.

파일 구조:
- css/variables.css (색상 변수)
- css/layout.css (레이아웃)
- css/components.css (컴포넌트 스타일)
- index.html

요구사항:
- sessionStorage에 로그인 정보 없으면 login.html로 리다이렉트
- 상단 고정 헤더: "동국제강 DK" 로고(남색 배경, DK는 주황), 오늘 날짜, 로그인 사용자명, 로그아웃 버튼
- 탭 네비게이션: 수급계획, 발주관리, 재고현황 (3개 탭)
- 탭 클릭 시 해당 콘텐츠 전환 (SPA 방식)
- 수급계획 탭에 KPI 카드 4개: 총 계획량, 국산 비율, 수입 비율, 평균 달성률
- 각 KPI 카드는 라벨/숫자/서브텍스트 구조
- 반응형 (900px 이하에서 1열)
- Chart.js 4.4.1 CDN 포함
- 디자인: 깔끔한 대시보드 스타일, 카드에 그림자, 둥근 모서리
```

**시연 포인트**:
- 한 번의 프롬프트로 CSS 3개 + HTML 1개 = 파일 4개가 한꺼번에 생성
- 브라우저에서 탭 전환, KPI 카드 확인
- "지금 프로그래머가 하루 걸리는 작업을 1분 만에 했습니다"

#### 2-2. 디자인 수정 요청 시연 (10분)

결과를 보고 수정 요청하는 과정을 보여줍니다:

```
KPI 카드의 숫자를 더 크게 만들어줘. 폰트 사이즈 36px.
두 번째 카드는 주황색 상단 라인을 추가해줘.
```

> **강사 멘트**: "바이브코딩의 핵심은 이 '대화'입니다. 결과를 보고 → 마음에 안 드는 걸 말로 → 고쳐달라고 합니다. 마치 디자이너에게 수정 요청하는 것과 같습니다."

### Step 3: 데이터 테이블 + 차트 (25분)

#### 3-1. 수급계획 테이블 + 차트 (15분)

```
수급계획 탭에 다음을 추가해줘.

1. 월별 계획 대비 실적 막대 차트 (Chart.js)
   - X축: 1월~12월
   - 데이터셋 2개: 계획(남색), 실적(주황)
   - 1~3월만 실적 있음, 나머지는 null

2. 월간 수급 계획 테이블
   - 컬럼: 월, 계획, 국산, 수입, 실적, 달성률, 상태
   - 12개월 샘플 데이터 포함
   - 달성률 95% 이상이면 초록 "달성" 배지, 85-95%면 주황 "미달" 배지, 실적 없으면 회색 "예정" 배지
   - 테이블 스타일: 줄무늬, 호버 효과

데이터는 js/data.js 파일에 배열로 분리해줘.
렌더링 로직은 js/plan.js에 만들어줘.
초기화는 js/app.js에서 해줘.
index.html 하단에 script 태그 순서: js/data.js → js/plan.js → js/app.js
```

**시연 포인트**:
- 차트가 나타나는 순간 — "우리가 매달 보는 수급계획 데이터가 차트로 자동 시각화됩니다"
- 배지 색상 — "달성/미달/예정이 자동으로 판정됩니다"

#### 3-2. 재고현황 탭 (10분)

```
재고현황 탭을 만들어줘.

- KPI 카드 3개: 전체 재고량, 인천 재고, 포항 재고
- 인천/포항 공장별 등급별 재고 막대 차트 2개 (현재 재고 vs 적정 재고)
- 재고 상세 테이블: 공장, 등급, 현재재고, 적정재고, 재고율, 상태(과잉/적정/부족), 수준 바
- 재고율 110% 이상 과잉(주황), 85-110% 적정(초록), 85% 미만 부족(빨강)
- 데이터는 js/data.js에 추가, 로직은 js/inventory.js
- 조회 전용 (편집 불가)
```

**시연 포인트**:
- "인천/포항 재고 데이터가 우리가 실제 관리하는 6개 등급별로 표시됩니다"
- 수준 바의 색상이 자동으로 바뀌는 것을 강조

### Step 4: 모달 폼 + 데이터 저장 (15분)

#### 4-1. 발주관리 탭 (15분)

```
발주관리 탭을 만들어줘.

1. KPI 카드 4개: 발주 건수, 발주 물량, 완료 건수, 진행 중 건수 (실제 데이터 기반 자동 계산)

2. 발주 목록 테이블
   - 컬럼: 발주번호, 거래처, 등급, 수량, 단가, 금액, 공장, 날짜, 상태
   - 샘플 데이터 10건 포함
   - 상태: 완료(초록), 진행(파랑), 대기(회색) 배지

3. 신규 발주 모달
   - "+ 신규 발주" 버튼 클릭 시 모달 오픈
   - 입력: 거래처(select), 등급(select), 수량, 단가, 공장(인천/포항), 납품일
   - "발주 등록" 클릭 시 목록 최상단에 추가, KPI 갱신, 모달 닫기
   - 발주번호는 PO-2026-NNNN 자동 생성

4. 데이터 저장
   - localStorage에 저장 (키: ordersData)
   - 페이지 새로고침해도 데이터 유지
   - 저장 형식: { data: [...], timestamp: "ISO날짜" }

js/order.js에 로직, js/data.js에 기본 데이터 추가해줘.
```

**시연 포인트**:
- 신규 발주 등록 → 목록에 즉시 반영 → KPI 숫자 변경 — "데이터가 실시간으로 연동됩니다"
- 새로고침 후에도 데이터 유지 — "브라우저에 자동 저장됩니다"

> **강사 멘트**: "지금 제가 한 일은 '요구사항 정의'입니다. 실무에서 기획서를 쓰는 것과 같습니다. 바이브코딩에서는 이 기획서가 곧 코드가 됩니다. 여러분이 업무 요구사항을 가장 잘 아는 사람들입니다."

### 앞면: 프롬프트 작성 요령

```
━━━ 바이브코딩 프롬프트 작성 요령 ━━━

✅ 좋은 프롬프트:
  "남색 배경에 주황 포인트 색상의 로그인 카드를
   중앙 정렬로 만들어줘. 아이디 2개 하드코딩."

❌ 나쁜 프롬프트:
  "로그인 페이지 만들어줘"

━━━ 핵심 규칙 ━━━

1. 구체적으로 쓸수록 좋다
   - 색상은 코드로 (#1a237e), 크기는 숫자로 (36px)
   - "예쁘게" 대신 "둥근 모서리, 그림자 효과"

2. 파일 구조를 지정한다
   - "login.html 하나에 다 넣어줘"
   - "CSS는 css/style.css로 분리해줘"

3. 데이터를 구체적으로 쓴다
   - "샘플 데이터 10건 포함"
   - "1~3월은 실적 있음, 나머지는 null"

━━━ 수정 요청 예시 ━━━

• "카드 간격을 더 넓혀줘"
• "글자 크기를 키워줘"
• "색상을 파란색에서 남색(#1a237e)으로 바꿔줘"
• "이 기능이 동작하지 않아. 에러 메시지는 ___야"
• "테이블에 호버 효과 추가해줘"
```
