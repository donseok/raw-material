# 바이브코딩 기초과정 강의 계획서

## 철스크랩 수급 계획 및 구매 관리 시스템 만들기

**대상**: 코딩 경험 없는 사내 임직원 50명
**시간**: 2시간
**강사**: 원료기획팀 (사내 강사)
**목표**: AI에게 한국어로 지시하여 대시보드 웹 사이트를 만들고 배포까지 완료

---

## 핵심 원칙

- **직접 타이핑 없음** — AI에게 한국어로 "이렇게 만들어줘"라고 설명하는 것이 전부
- **완성 경험 우선** — 2시간 안에 배포까지 완료해서 "내가 만든 사이트"를 가져감
- **이 프로젝트를 축소 재현** — 전체가 아닌 핵심 3개 탭(수급계획, 발주관리, 재고현황)

---

## 사전 준비 (강의 전)

### 수강생 PC 세팅 (필수)

| 항목 | 내용 |
|------|------|
| **Cursor IDE** | https://cursor.com 설치 (무료 플랜으로 충분) |
| **Chrome 브라우저** | 최신 버전 |
| **GitHub 계정** | https://github.com 가입 |
| **Vercel 계정** | https://vercel.com GitHub 연동 가입 |

> **Cursor를 추천하는 이유**: Claude/GPT가 내장되어 있고, 채팅창에 한국어로 지시하면 파일을 자동 생성/수정합니다. 별도 복붙 불필요.

### 강사 준비물

- 완성된 이 프로젝트의 **라이브 데모** (동기부여용)
- 각 단계별 **프롬프트 치트시트** (인쇄물 or 공유 문서)
- 단계별 **체크포인트 코드** (GitHub에 브랜치별로 준비 — 뒤처지는 수강생 구제용)

## Step 0: 오프닝 (10분)

### 완성작 시연 (5분)

이 프로젝트의 라이브 데모를 보여주며 동기부여합니다.

> "이 사이트를 오늘 2시간 안에 여러분이 직접 만듭니다. 코드를 한 줄도 직접 쓰지 않습니다."

### 바이브코딩이란? (5분)

> "프로그래밍 언어를 몰라도 됩니다. AI에게 한국어로 '이렇게 만들어줘'라고 말하면, AI가 코드를 만들어줍니다. 여러분의 역할은 **무엇을 만들지 설명하는 것**과 **결과를 확인하고 수정 요청하는 것**입니다."

핵심 사이클 설명:

```
① 한국어로 지시 → ② AI가 코드 생성 → ③ 브라우저에서 확인 → ④ 수정 요청
```

## Step 1: 프로젝트 생성 + 로그인 페이지 (15분)

### 1-1. Cursor에서 프로젝트 폴더 생성 (3분)

강사가 화면 공유하며 같이 진행:

1. Cursor 실행
2. File → Open Folder → 바탕화면에 `my-dashboard` 폴더 생성 후 열기

### 1-2. 첫 번째 프롬프트 (12분)

Cursor 채팅창(Cmd+L / Ctrl+L)에 입력:

```
동국제강 철스크랩 관리 시스템의 로그인 페이지를 만들어줘.

파일: login.html (단일 HTML 파일, CSS/JS 인라인)

요구사항:
- 남색(#1a237e) + 주황(#ff8f00) 컬러 테마
- 중앙 정렬 로그인 카드
- 아이디/비밀번호 입력 필드
- 로그인 버튼
- 계정 2개 하드코딩: dongkuk1/1234 (김철호 과장), dongkuk2/1234 (박영수 대리)
- 로그인 성공 시 sessionStorage에 사용자 정보 저장 후 index.html로 이동
- 로그인 실패 시 빨간색 오류 메시지 + 흔들기 애니메이션
- 이미 로그인된 상태면 index.html로 자동 이동
- 한국어 UI
```

**수강생 실습**: 프롬프트를 복붙 → Accept 클릭 → login.html 생성됨

**확인**: 파일 우클릭 → "Open in Browser" 또는 `npx serve .`

> **강사 포인트**: "프롬프트가 곧 설계서입니다. 구체적으로 쓸수록 원하는 결과가 나옵니다."

## Step 2: 대시보드 뼈대 (25분)

### 2-1. 기본 구조 프롬프트 (15분)

```
이제 메인 대시보드 index.html을 만들어줘.

파일 구조:
- css/variables.css (색상 변수)
- css/layout.css (레이아웃)
- css/components.css (컴포넌트 스타일)
- index.html

요구사항:
- sessionStorage에 로그인 정보 없으면 login.html로 리다이렉트
- 상단 고정 헤더: "동국제강 DK" 로고(남색 배경, DK는 주황), 오늘 날짜, 로그인 사용자명, 로그아웃 버튼
- 탭 네비게이션: 수급계획, 발주관리, 재고현황 (3개 탭)
- 탭 클릭 시 해당 콘텐츠 전환 (SPA 방식)
- 수급계획 탭에 KPI 카드 4개: 총 계획량, 국산 비율, 수입 비율, 평균 달성률
- 각 KPI 카드는 라벨/숫자/서브텍스트 구조
- 반응형 (900px 이하에서 1열)
- Chart.js 4.4.1 CDN 포함
- 디자인: 깔끔한 대시보드 스타일, 카드에 그림자, 둥근 모서리
```

**수강생 실습**: 프롬프트 입력 → 파일 4개 생성됨 → 브라우저 확인

### 2-2. 디자인 수정 요청 연습 (10분)

결과를 보고 수정 요청하는 연습을 합니다. 예시:

```
KPI 카드의 숫자를 더 크게 만들어줘. 폰트 사이즈 36px.
두 번째 카드는 주황색 상단 라인을 추가해줘.
```

> **강사 포인트**: "바이브코딩의 핵심은 이 '대화'입니다. 결과를 보고 → 마음에 안 드는 걸 말로 → 고쳐달라고 하면 됩니다."

## Step 3: 데이터 테이블 + 차트 (25분)

### 3-1. 수급계획 테이블 + 차트 (15분)

```
수급계획 탭에 다음을 추가해줘.

1. 월별 계획 대비 실적 막대 차트 (Chart.js)
   - X축: 1월~12월
   - 데이터셋 2개: 계획(남색), 실적(주황)
   - 1~3월만 실적 있음, 나머지는 null

2. 월간 수급 계획 테이블
   - 컬럼: 월, 계획, 국산, 수입, 실적, 달성률, 상태
   - 12개월 샘플 데이터 포함
   - 달성률 95% 이상이면 초록 "달성" 배지, 85-95%면 주황 "미달" 배지, 실적 없으면 회색 "예정" 배지
   - 테이블 스타일: 줄무늬, 호버 효과

데이터는 js/data.js 파일에 배열로 분리해줘.
렌더링 로직은 js/plan.js에 만들어줘.
초기화는 js/app.js에서 해줘.
index.html 하단에 script 태그 순서: js/data.js → js/plan.js → js/app.js
```

### 3-2. 재고현황 탭 (10분)

```
재고현황 탭을 만들어줘.

- KPI 카드 3개: 전체 재고량, 인천 재고, 포항 재고
- 인천/포항 공장별 등급별 재고 막대 차트 2개 (현재 재고 vs 적정 재고)
- 재고 상세 테이블: 공장, 등급, 현재재고, 적정재고, 재고율, 상태(과잉/적정/부족), 수준 바
- 재고율 110% 이상 과잉(주황), 85-110% 적정(초록), 85% 미만 부족(빨강)
- 데이터는 js/data.js에 추가, 로직은 js/inventory.js
- 조회 전용 (편집 불가)
```

## Step 4: 모달 폼 + 데이터 저장 (20분)

### 4-1. 발주관리 탭 (20분)

```
발주관리 탭을 만들어줘.

1. KPI 카드 4개: 발주 건수, 발주 물량, 완료 건수, 진행 중 건수 (실제 데이터 기반 자동 계산)

2. 발주 목록 테이블
   - 컬럼: 발주번호, 거래처, 등급, 수량, 단가, 금액, 공장, 날짜, 상태
   - 샘플 데이터 10건 포함
   - 상태: 완료(초록), 진행(파랑), 대기(회색) 배지

3. 신규 발주 모달
   - "+ 신규 발주" 버튼 클릭 시 모달 오픈
   - 입력: 거래처(select), 등급(select), 수량, 단가, 공장(인천/포항), 납품일
   - "발주 등록" 클릭 시 목록 최상단에 추가, KPI 갱신, 모달 닫기
   - 발주번호는 PO-2026-NNNN 자동 생성

4. 데이터 저장
   - localStorage에 저장 (키: ordersData)
   - 페이지 새로고침해도 데이터 유지
   - 저장 형식: { data: [...], timestamp: "ISO날짜" }

js/order.js에 로직, js/data.js에 기본 데이터 추가해줘.
```

> **강사 포인트**: "지금 여러분이 한 일은 '요구사항 정의'입니다. 실무에서 기획서를 쓰는 것과 같습니다. 바이브코딩에서는 이 기획서가 곧 코드가 됩니다."

## Step 5: 배포 (15분)

### 5-1. GitHub 업로드 (7분)

Cursor 터미널(Ctrl+`)에서:

```bash
git init
git add .
git commit -m "내 첫 번째 바이브코딩 프로젝트"
```

GitHub에서 새 저장소 생성 → Cursor에서:

```bash
git remote add origin https://github.com/본인계정/my-dashboard.git
git push -u origin main
```

> 이 과정이 어려운 수강생을 위해 **강사가 화면 공유하며 한 단계씩** 진행합니다.

### 5-2. Vercel 배포 (5분)

1. https://vercel.com 접속
2. "Add New Project"
3. GitHub 저장소 선택
4. Deploy 클릭
5. 30초 후 → **나만의 URL 완성!**

### 5-3. 마무리 (3분)

> "여러분의 휴대폰에서 이 URL을 열어보세요. 여러분이 2시간 전까지는 코딩을 한 번도 해본 적 없는 분들이었습니다. 지금 여러분은 로그인, 대시보드, 차트, 데이터 저장까지 되는 웹 사이트를 만들고, 전 세계에서 접속 가능한 주소로 배포까지 완료했습니다."

## 프롬프트 치트시트 (인쇄물)

수강생에게 나눠줄 A4 1장 분량:

```
━━━ 바이브코딩 프롬프트 작성 요령 ━━━

✅ 좋은 프롬프트:
  "남색 배경에 주황 포인트 색상의 로그인 카드를
   중앙 정렬로 만들어줘. 아이디 2개 하드코딩."

❌ 나쁜 프롬프트:
  "로그인 페이지 만들어줘"

━━━ 수정 요청 예시 ━━━

• "카드 간격을 더 넓혀줘"
• "글자 크기를 키워줘"
• "색상을 파란색에서 남색(#1a237e)으로 바꿔줘"
• "이 기능이 동작하지 않아. 에러 메시지는 ___야"
• "테이블에 호버 효과 추가해줘"

━━━ 문제 해결 ━━━

• 화면이 안 나오면 → F12 → Console 탭 → 빨간 글자 복사 → AI에게 보여주기
• 파일이 안 보이면 → Cursor 왼쪽 파일 탐색기 확인
• 기능이 안 되면 → "이 부분이 안 돼. 원하는 동작은 ___야" 라고 설명
```
