# 프로세스 정의서

## 철스크랩 수급 계획 및 구매 관리 시스템

**동국제강 주식회사 원료기획팀**
**시스템 버전**: 1.0
**최종 수정일**: 2026년 3월

---

## 1. 업무 프로세스 전체 흐름

```
[수급계획 수립] → [공장배분] → [발주/수입계약] → [재고관리] → [구매실적 분석]
     ↑                                                           ↓
     └──────────── 실적 피드백 반영 ────────────────────────────┘
```

### 1.1 프로세스 단계 요약

| 단계 | 프로세스 | 입력 | 출력 | 모듈 |
|------|----------|------|------|------|
| 1 | 수급계획 수립 | 연간 수요 예측, 국산/수입 비율 | 월별 계획량 (12개월) | plan.js |
| 2 | 공장배분 | 수급계획, 공장별 생산능력 | 인천/포항 배분량 | alloc.js |
| 3 | 국내 발주 | 배분량, 거래처 정보 | 발주서 (PO) | order.js |
| 4 | 수입 계약 | 수입 물량 계획 | 수입 계약서 (IMP) | import.js |
| 5 | 거래처 관리 | 거래처 납품 실적 | 신뢰등급, 공급능력 평가 | supplier.js |
| 6 | 재고 모니터링 | 입고량, 소비량 | 재고 상태 판단 | inventory.js |
| 7 | 구매실적 분석 | 월별 구매 데이터 | 통계, CSV 리포트 | stats.js |
| 8 | 대시보드 문서 내보내기 | 전체 탭 화면 | DOCX 보고서 파일 | export-doc.js |

## 2. 수급계획 수립 프로세스 (P-01)

### 2.1 프로세스 개요

| 항목 | 내용 |
|------|------|
| 프로세스 ID | P-01 |
| 프로세스명 | 연간 수급계획 수립 |
| 담당자 | 원료기획팀 과장 |
| 주기 | 연 1회 수립, 월 1회 실적 반영 |
| 관련 모듈 | plan.js |

### 2.2 상세 흐름

```
시작
 │
 ├─① 연간 총 수요량 산정
 │    - 전년도 실적 기반
 │    - 생산계획 연동
 │
 ├─② 국산/수입 비율 설정
 │    - 슬라이더로 비율 조정 (기본: 국산 80%, 수입 20%)
 │    - 월간 목표 물량 설정 (기본: 225,000톤/월)
 │
 ├─③ 월별 계획량 배정
 │    - 12개월 계획/국산/수입 물량 입력
 │    - 셀 편집 또는 엑셀 붙여넣기
 │
 ├─④ 실적 반영 및 달성률 확인
 │    - 실적 데이터 입력 (actual)
 │    - 자동 달성률 계산: (실적 ÷ 계획) × 100
 │
 └─⑤ KPI 확인 및 계획 조정
      - 총 계획량, 국산/수입 비율, 평균 달성률 모니터링
```

### 2.3 데이터 모델

```javascript
// 수급계획 1행 (월별)
{
  plan: Number,    // 계획량 (톤, ≥ 0, 필수)
  dom: Number,     // 국산 물량 (톤, ≥ 0, 필수)
  imp: Number,     // 수입 물량 (톤, ≥ 0, 필수)
  actual: Number|null  // 실적 (톤, ≥ 0, null 허용)
}
```

### 2.4 달성률 판정 기준

| 달성률 | 상태 | 후속 조치 |
|--------|------|----------|
| 95% 이상 | 달성 | 계획 유지 |
| 85% ~ 95% | 미달 | 원인 분석, 발주 조정 |
| 85% 미만 | 부족 | 긴급 추가 발주 검토 |

## 3. 공장배분 프로세스 (P-02)

### 3.1 프로세스 개요

| 항목 | 내용 |
|------|------|
| 프로세스 ID | P-02 |
| 프로세스명 | 공장별 물량 배분 |
| 담당자 | 원료기획팀 |
| 주기 | 월 1회 |
| 관련 모듈 | alloc.js |

### 3.2 상세 흐름

```
시작
 │
 ├─① 수급계획에서 월별 총 물량 확인
 │
 ├─② 인천/포항 공장 배분 비율 결정
 │    - 생산능력, 재고 수준, 수요 기반
 │    - 기본 비율: 인천 60%, 포항 40%
 │
 ├─③ 월별 배분 계획 입력
 │    - 인천 계획(ic), 포항 계획(pc)
 │
 ├─④ 실적 입력 및 달성률 추적
 │    - 인천 실적(ia), 포항 실적(pa)
 │    - 달성률 = (실적 ÷ 계획) × 100
 │
 └─⑤ 현황 카드 및 차트 확인
      - 도넛 차트: 실적 vs 미달 비율
      - 진행 바: 전체 기간 달성률
```

### 3.3 데이터 모델

```javascript
// 공장배분 1행 (월별)
{
  ic: Number,  // 인천 계획 (톤, ≥ 0, 필수)
  ia: Number,  // 인천 실적 (톤, ≥ 0, 필수)
  pc: Number,  // 포항 계획 (톤, ≥ 0, 필수)
  pa: Number   // 포항 실적 (톤, ≥ 0, 필수)
}
```

## 4. 발주관리 프로세스 (P-03)

### 4.1 프로세스 개요

| 항목 | 내용 |
|------|------|
| 프로세스 ID | P-03 |
| 프로세스명 | 국내 철스크랩 발주 |
| 담당자 | 원료기획팀 대리/과장 |
| 주기 | 수시 |
| 관련 모듈 | order.js |

### 4.2 상세 흐름

```
시작
 │
 ├─① 발주 필요량 확인
 │    - 공장배분 계획 기반
 │    - 재고 현황 참조
 │
 ├─② 거래처 선정
 │    - 거래처 목록에서 공급능력, 신뢰등급 확인
 │    - 지역 및 공장 접근성 고려
 │
 ├─③ 발주 등록
 │    - 거래처, 등급, 수량, 매입단가, 납품공장, 납품일 입력
 │    - 발주번호 자동 생성: PO-YYYY-NNNN
 │    - 초기 상태: '대기'
 │
 ├─④ 발주 진행 상태 추적
 │    - 대기 → 진행 → 완료
 │
 ├─⑤ 발주서 출력
 │    - PDF 미리보기 → 인쇄 또는 PDF 저장
 │
 └─⑥ KPI 모니터링
      - 발주 건수, 물량, 완료/진행 건수
```

### 4.3 데이터 모델

```javascript
{
  no: String,      // 발주번호 "PO-2026-0341"
  sup: String,     // 거래처명
  grade: String,   // 등급 (생철/중량/경량/길로틴/선반/압축)
  qty: Number,     // 수량 (톤, ≥ 0)
  price: Number,   // 매입단가 (원/톤, ≥ 0)
  plant: String,   // 납품공장 ("인천" | "포항")
  date: String,    // 납품일 "YYYY-MM-DD"
  status: String   // 상태 ("완료" | "진행" | "대기")
}
```

### 4.4 발주 상태 전이

```
대기 ──→ 진행 ──→ 완료
```

| 상태 | 의미 | 배지 색상 |
|------|------|----------|
| 대기 | 발주 등록, 미착수 | 회색 |
| 진행 | 출하/운송 중 | 파랑 |
| 완료 | 납품 완료 | 초록 |

```
시작
 │
```

```javascript
{

```

### 5.4 수입계약 상태 전이

```
계약 ──→ 선적 ──→ 운송중 ──→ 도착
```

| 상태 | 의미 | 배지 색상 |
|------|------|----------|

## 6. 거래처관리 프로세스 (P-05)

### 6.1 프로세스 개요

| 항목 | 내용 |
|------|------|
| 프로세스 ID | P-05 |
| 프로세스명 | 거래처 등록 및 평가 |
| 담당자 | 원료기획팀 |
| 주기 | 수시 등록, 연 1회 평가 |
| 관련 모듈 | supplier.js |

### 6.2 상세 흐름

```
시작
 │
 ├─① 거래처 등록
 │    - 거래처명, 대표자, 지역, 연락처, 월 공급능력
 │    - 코드 자동 생성: S001, S002, ...
 │    - 신규 등록 시 등급: '신규', 신뢰도: 0%
 │
 ├─② 납품 실적 누적
 │    - 발주 완료 시 연간 납품량(ytd) 갱신
 │
 ├─③ 신뢰등급 평가
 │    - A: 신뢰도 90% 이상
 │    - B+: 신뢰도 85% 이상
 │    - B: 신뢰도 85% 미만
 │
 └─④ 차트 기반 분석
      - 파이 차트: 거래처별 납품 비중
      - 라인 차트: 주요 거래처 월별 납품 추이
```

### 6.3 데이터 모델

```javascript
{
  code: String,    // 거래처코드 "S001"
  name: String,    // 거래처명
  region: String,  // 지역
  rep: String,     // 대표자
  tel: String,     // 연락처
  cap: Number,     // 월 공급능력 (톤, ≥ 0)
  ytd: Number,     // 연간 누적 납품량 (톤, ≥ 0)
  grade: String,   // 신뢰등급 (A/B+/B/신규)
  rate: Number     // 신뢰도 (0~100)
}
```

```
재고율 = (현재재고 ÷ 적정재고) × 100

```

```javascript
  opt: Number     // 적정 재고 (톤)
}
```

## 8-1. 대시보드 문서 내보내기 프로세스 (P-08)

### 8-1.1 프로세스 개요

| 항목 | 내용 |
|------|------|
| 프로세스 ID | P-08 |
| 프로세스명 | 대시보드 DOCX 내보내기 |
| 담당자 | 전체 사용자 |
| 주기 | 수시 (보고 필요 시) |
| 관련 모듈 | export-doc.js |

### 8-1.2 상세 흐름

```
시작
 │
 ├─① DOC 저장 버튼 클릭
 │    - 버튼 비활성화, 진행 표시
 │    - Chart.js 애니메이션 비활성화
 │
 ├─② 탭 순회 및 캡처
 │    - 7개 탭 순서대로 활성화
 │    - 각 탭 차트 재렌더링 (350ms 대기)
 │    - html2canvas로 각 블록 이미지 캡처
 │
 ├─③ DOCX 문서 생성
 │    - docx 라이브러리로 Word 문서 구성
 │    - 탭별 섹션 (페이지 나눔)
 │    - 표지: 제목, 생성 일시, 사용자명
 │
 ├─④ 파일 다운로드
 │    - FileSaver.js로 .docx 다운로드
 │    - 파일명: steel-scrap-dashboard-YYYY-MM-DD.docx
 │
 └─⑤ 원래 상태 복원
      - 원래 활성 탭으로 복귀
      - Chart.js 애니메이션 복원
      - 버튼 재활성화
```

### 8-1.3 사용 라이브러리

## 9. 공통 프로세스

### 9.1 인증 프로세스

```
사용자 접속
 │
 ├─ sessionStorage에 loggedInUser 존재?
 │   ├─ Yes → index.html 대시보드 표시
 │   └─ No → login.html 리다이렉트
 │
 ├─ 아이디/비밀번호 입력
 │   ├─ 일치 → sessionStorage 저장 → 대시보드 이동
 │   └─ 불일치 → 오류 메시지 표시
 │
 └─ 로그아웃
     └─ sessionStorage 삭제 → login.html 이동
```

### 9.2 데이터 저장/복원 프로세스

```
데이터 편집
 │
 ├─ normalize*() 함수로 입력값 검증
 │   ├─ 유효 → 전역 변수 갱신 → writeStoredData() → localStorage 저장
 │   └─ 무효 → showToast() 오류 알림 → 변경 취소
 │
 └─ 페이지 로드 시
     ├─ readStoredData()로 localStorage 읽기
     │   ├─ 유효 → 전역 변수에 복원
     │   └─ 무효 → localStorage 키 삭제 → 기본값(default*Data) 사용
     └─ init*() 호출 → 화면 렌더링
```

### 9.3 데이터 저장 형식

```javascript
// localStorage 저장 형식 (공통)
{
  "data": [...],                      // 실제 데이터 배열
  "timestamp": "2026-03-10T09:00:00Z" // 저장 시각 (ISO 8601)
}
```

### 9.4 초기화 프로세스

```
초기화 버튼 클릭
 │
 ├─ confirm() 확인 팝업 표시
 │   ├─ 승인 → localStorage.removeItem(key) → cloneData(default*Data) → 재렌더링
 │   └─ 취소 → 아무 작업 없음
 │
 └─ showToast() 완료 알림
```
